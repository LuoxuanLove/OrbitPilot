# 扩展指南

## 开发基线

当前扩展应以以下事实为准：

- UI 主控制器当前在 `plugin.gd` 的内联 `DockController`
- `ui/orbit_dock.tscn` 当前为纯 UI 场景
- `ui/orbit_dock.gd` / `ui/orbit_dock_controller.gd` 不是当前主运行路径

如果扩展 UI，请先判断是否继续沿用“纯场景 + 内联控制器”路径，还是要重新恢复独立控制器脚本。未经验证不要混用两种模式。

## 新增本地工具

1. 在 `core/tools/local_tools.gd` 增加 `get_definitions()` 项
2. 在 `execute()` 增加 action 分发
3. 若涉及写入，必须走 `ChangeSetService`
4. 在 `docs/模块/本地工具与上下文索引.md` 更新说明
5. 更新 `docs/测试/测试策略.md` 中对应测试项

## 新增远端服务器

1. 在设置中增加 `remote_servers.{id}`
2. 确认 endpoint 填写的是根地址，而不是 `/mcp`
3. 运行“刷新远端工具”
4. 确认工具目录中出现 `remote:{id}:*`
5. 更新 `docs/模块/远端MCP互通.md`

## 新增 Provider

1. 新建 `core/provider/{xxx}_provider.gd` 继承 `ProviderAdapter`
2. 实现 `get_id()` 与 `send_chat()`，必要时实现 `send_chat_stream()`
3. 在 `plugin.gd` 注册实例到 `_execution_service.setup(...)`
4. 在默认设置中补齐 provider 配置
5. 在 UI 的 provider 选择器中展示
6. 更新：
   - `docs/模块/Provider层.md`
   - `docs/测试/测试策略.md`

## 新增 OpenAI Compatible 预设 Provider

例如 DeepSeek 这一类，只需：

1. 在 `core/types.gd` 添加 provider 配置
2. 在 `plugin.gd` 的 provider map 中映射到 `_openai_provider`
3. 补充默认 `base_url`、`model`、`api_key_env`
4. 更新 `Provider层.md`

这种路径适合：

- 协议兼容 OpenAI
- 只需差异化默认配置
- 不需要独立的响应解析逻辑

## 扩展 Sessions 页面

当前 Sessions 页包含：

- Active 列表
- Archived 列表
- New Session
- Open Session
- Archive Selected
- Delete Archived
- Rename

若继续扩展：

1. 先补 `_settings["sessions"]` 数据字段
2. 再补 `DockController.apply_model()` 渲染逻辑
3. 再补对应 `_on_session_*` 主控处理函数
4. 最后补文档与测试

## 扩展导航页

新增页面时至少同步修改：

- `ui/orbit_dock.tscn` 页面节点结构
- `DockController.PAGE_*` 常量
- `_assign_nodes()`
- `_bind_events()`
- `_set_page()`
- `_apply_nav_state()`

## 文档同步要求

任何涉及以下内容的改动，都必须同步更新文档：

- 运行时入口变更
- Provider 接入方式变更
- MCP endpoint 约定变更
- 会话数据结构变更
- ChangeSet 审批链变更
- UI 页面结构变更
