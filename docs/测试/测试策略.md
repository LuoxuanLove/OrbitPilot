# 测试策略

## 1. 安装与启动

目标：确认插件独立可用。

检查项：

- 仅安装 `addons/orbit_pilot`
- 插件启用后 Dock 正常显示
- 未配置远端 MCP 时，本地工具仍可用
- Sessions / Tools / Review / Settings 页面可切换

## 2. 最小自动回归

入口：

- Tools 页面执行 `local:run_regression_suite`

预期：

- 返回 `total / passed / failed / items[]`
- 若存在失败 case，应先修复基础能力再做集成测试

## 3. Provider 联通

### OpenAI Compatible

- 正常返回 assistant 内容
- 流式场景有 `message_delta`
- 错误 API Key 时返回结构化错误

### DeepSeek

- 选择 `deepseek` 后，执行链真正走 OpenAI Compatible Provider
- `base_url` 应为 `https://api.deepseek.com`
- `model` 应为 `deepseek-chat`
- 可成功返回中文回复

### Anthropic

- 正常返回 assistant 内容
- 工具调用结构可被执行层消费

## 4. 远端 MCP

### 正常路径

- endpoint 填写根地址，例如 `http://127.0.0.1:3000`
- 健康检查成功
- `GET /api/tools` 成功
- 工具目录出现 `remote:{id}:*`
- 可正常调用至少一个只读远端工具

### 错误路径

- endpoint 填写为 `http://127.0.0.1:3000/mcp`
- 预期会请求 `/mcp/health` 和 `/mcp/api/tools`
- 通常返回 `404`
- 文档与 UI 应明确提示用户配置错误

## 5. 执行链策略

### analyze

- 不执行写入
- 不生成 applied 结果

### propose

- 允许读取工具
- 写入类工具被拦截或只产出提案

### apply_with_approval

- 可生成 pending change set
- 必须进入 Review 审批后才能落盘

## 6. ChangeSet

检查项：

- 创建 change set 成功
- Review 页面可查看 diff
- 审批通过后文件落盘
- 审批拒绝后状态为 rejected
- `rollback_last` 可恢复最近一次 applied 内容

## 7. 会话管理

检查项：

- Sessions 页面可打开
- Active / Archived 分组显示正确
- `Open Session` 可切换当前会话
- `Archive Selected` 仅归档选中会话
- 被归档的当前会话会触发新 active session 创建
- Archived 会话可删除
- 任意会话可重命名
- 聊天页会话选择器不再显示 archived 会话

## 8. 流式输出与取消

检查项：

- 长回复时出现 `message_delta`
- 回合完成后正式 assistant 回复只显示一次
- 取消后 active run 状态退出
- 取消后不再继续追加 preview

## 9. 工具页与 Review 页

检查项：

- Tools 页可手动执行本地工具
- Tools 页能显示结构化结果
- Review 页可切换 pending change set
- Review 页审批动作能反映到状态与日志
