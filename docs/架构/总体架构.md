# 总体架构

## 当前运行时分层

```text
EditorPlugin (plugin.gd)
  ├─ DockController (inner class in plugin.gd)
  │   └─ Workbench UI (ui/orbit_dock.tscn)
  ├─ Provider Layer
  │   ├─ OpenAIProvider
  │   └─ AnthropicProvider
  ├─ Tool Layer
  │   ├─ LocalTools
  │   ├─ RemoteMcpClient
  │   ├─ ToolCatalog
  │   └─ ToolRouter
  ├─ Context Layer
  │   └─ ContextIndexService
  ├─ Execution Layer
  │   └─ ExecutionService
  ├─ Change Layer
  │   └─ ChangeSetService
  └─ Test Layer
      └─ OrbitRegressionRunner
```

## 关键事实

### 1. `plugin.gd` 是装配根

`plugin.gd` 当前负责：

- 初始化所有服务对象。
- 维护插件状态：`_settings`、`_logs`、`_validation_summary`、`_active_run_id` 等。
- 创建并刷新 Dock。
- 处理 Provider 请求、工具执行、审批动作、会话管理。
- 内联 `DockController`，避免场景根脚本实例化链不稳定。

### 2. `ui/orbit_dock.tscn` 当前是纯 UI 场景

场景只负责：

- 节点结构。
- 页面布局。
- 输入控件、列表、按钮、文本面板。

场景当前不承担：

- 网络访问。
- 会话状态管理。
- Provider 调用。
- 工具执行。

### 3. `DockController` 当前内联在 `plugin.gd`

原因：

- 早期尝试过 `ui/orbit_dock.gd` 和独立 `ui/orbit_dock_controller.gd`。
- 在 Godot 编辑器插件环境下，根脚本与独立控制器脚本都出现过 placeholder / 实例化不稳定问题。
- 当前实现把 UI 控制器直接内联到 `plugin.gd`，作为当前最稳定路径。

## 主数据流

### 一次典型回合

1. 用户在聊天页输入 prompt。
2. `DockController` 把动作回调给 `plugin.gd`。
3. `plugin.gd`：
   - 取当前 active session
   - 追加 user message
   - 组装 provider 配置
   - 组装统一工具目录
4. `ExecutionService.execute_turn(...)` 调用 Provider。
5. Provider 可能返回：
   - 直接 assistant 文本
   - tool_calls
   - 流式增量事件
6. `ExecutionService` 通过 `ToolRouter` 调用工具。
7. 若工具产生写入，则通过 `ChangeSetService` 生成 pending change set。
8. UI 在 Review 页面展示 diff 与审批操作。
9. 审批通过后才真正落盘。
10. 回合完成后会更新日志、会话消息、工具结果与状态栏。

## 会话数据流

当前会话状态保存在 `_settings["sessions"]` 中，每条会话包含：

- `id`
- `title`
- `messages`
- `run_records`
- 可选 `archived`
- 可选 `archived_at`

当前逻辑约束：

- `last_session_id` 优先选择未归档会话。
- 若当前会话被归档且没有其他 active 会话，会自动创建一个新的默认会话。
- 归档会话不会再出现在聊天页会话选择器中，只在 Sessions 页面展示。

## 失败与降级策略

- Provider 请求失败：在当前会话中追加 `Request failed: ...`。
- Remote MCP 不可达：远端工具从目录中移除，仅保留本地工具。
- Tool 执行失败：写日志并返回结构化错误，不导致插件退出。
- 流式失败：非取消场景下允许回退非流式一次。
- Dock UI 控制失败：优先以 `plugin.gd` 内联控制器为唯一运行路径。

## 需要特别注意的遗留实现

仓库中仍保留：

- `ui/orbit_dock.gd`
- `ui/orbit_dock_controller.gd`

当前它们不是主运行路径。

后续若要正式清理或恢复独立控制器模式，需要先完成：

1. 引用链审计。
2. 场景脚本绑定确认。
3. 编辑器插件模式下的实例化稳定性验证。
