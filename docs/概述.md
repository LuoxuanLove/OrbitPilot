# 概述

## 产品定位

`OrbitPilot` 是运行在 Godot 编辑器内部的独立 Agent 插件。

目标：

- 提供类 Codex / Claude Code 的编辑器内开发体验。
- 通过可选远端 MCP 接入外部工具能力，但不与任何外部 Godot 插件形成源码依赖。
- 强制把写入行为收敛到审批链中，避免无授权落盘。

## 当前版本边界

当前仓库实现已经覆盖：

- 独立插件入口、独立设置文件、独立 Dock 工作台。
- Provider 适配：OpenAI Compatible、DeepSeek、Anthropic。
- Remote MCP Client：HTTP `GET /health`、`GET /api/tools`、`POST /mcp`。
- Local Tools：
  - `search_files`
  - `search_text`
  - `read_file`
  - `get_editor_context`
  - `get_script_symbols`
  - `get_scene_dependencies`
  - `get_context_bundle`
  - `run_regression_suite`
  - `create_change_set`
  - `get_change_set`
  - `apply_change_set`
- Context Index：脚本符号索引、场景依赖索引。
- 执行策略：`analyze`、`propose`、`apply_with_approval`。
- Sessions 页面：
  - Active / Archived 分组
  - 打开会话
  - 新建会话
  - 归档当前会话
  - 删除已归档会话
  - 重命名任意会话
- 最小自动回归：`local:run_regression_suite`。

## 当前运行时入口

当前实际运行时路径：

- `plugin.gd`：服务装配根、DockController、主状态管理。
- `ui/orbit_dock.tscn`：纯 UI 场景结构。

注意：

- `DockController` 当前内联在 `plugin.gd` 中；后续重构将其提取到 `ui/dock_controller.gd` 并拆分为 5 个页面控制器。
- 文档、调试和功能扩展以 `plugin.gd` 内部的 `DockController` 为准。

## 非目标

当前版本明确不做：

- OrbitPilot 自身对外提供 MCP Server。
- 覆盖 Godot 全量编辑器 API。
- 无人值守连续写入。
- 在插件启动时强依赖远端 MCP 可达。

## 核心约束

- 所有写入必须先创建 `ChangeSet`，再审批应用。
- 远端 MCP 失败时必须自动降级，不阻塞插件启动与本地工具使用。
- 工具 ID 必须带来源前缀，避免同名冲突。
- Remote MCP endpoint 必须填写服务根地址，不允许直接填写 `/mcp`。
- Provider 切换必须真正影响执行链，而不是只更新 UI 显示。

## 当前推荐调试顺序

1. 先确认插件可启动，Dock 可显示。
2. 先跑 `local:run_regression_suite`。
3. 再验证 Provider 联通。
4. 再验证 Remote MCP 健康检查和工具发现。
5. 最后验证 `prompt -> tool -> change set -> approval` 闭环。
