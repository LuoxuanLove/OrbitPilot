# 流式通道

## 目标

通过 SSE（Server-Sent Events）实现模型增量输出，降低长回复首字延迟，并允许用户主动取消。

## 公共流式实现

文件：`core/provider/provider_adapter.gd`

核心公共能力：

- `_stream_sse_json(...)`
- `_drain_sse_buffer(...)`

能力边界：

- 建立 `HTTPClient` 连接
- 发起 `POST`
- 按 SSE 边界解析 `data:`
- 处理 `[DONE]`
- 把 JSON 片段交给 Provider 专属解析器

## OpenAI Compatible 流式

文件：`core/provider/openai_provider.gd`

当前路径：

- Endpoint: `/chat/completions`
- `stream = true`

当前增量来源：

- `choices[0].delta.content` -> `message_delta`
- `choices[0].delta.tool_calls` -> `tool_call_delta`

DeepSeek 当前也走这一路径。

## Anthropic 流式

文件：`core/provider/anthropic_provider.gd`

当前路径：

- Endpoint: `/v1/messages`
- `stream = true`

当前增量来源：

- `text_delta` -> `message_delta`
- `input_json_delta` -> `tool_call_delta`

## 取消语义

当前取消链：

1. UI 点击 `Cancel`
2. 主控调用 `ExecutionService.cancel_run(run_id)`
3. Provider 在连接、轮询、chunk 处理阶段检查取消回调
4. 命中取消时立即关闭连接并返回 `cancelled`
5. 执行层映射为 `run_cancelled`

## 回退策略

- 流式失败且不是取消时，执行层允许回退到非流式一次
- 取消场景不允许回退，避免用户已经取消后请求仍继续执行

## 当前注意点

### 1. 流式成功后必须清 preview

否则 UI 可能重复显示 assistant 内容。

### 2. OpenAI Compatible 工具名不要直接用带前缀的完整 ID

原因：

- 某些兼容服务会对函数名字符做限制
- `local:search_files` 这类名称可能触发 `400`

当前处理：

- 对模型暴露短工具名
- 完整工具 ID 放到 description

### 3. HTTP 错误要保留 response_code 和 body

原因：

- 仅显示 `http_error_status` 不足以定位问题
- 失败时应保留厂商返回的 `message` 或 `detail`
