## 工具分类

### 读取类工具

- `search_files`
- `search_text`
- `read_file`
- `get_editor_context`
- `get_script_symbols`
- `get_scene_dependencies`
- `get_context_bundle`

用途：

- 项目结构检索
- 当前编辑器上下文采集
- 单文件与符号级理解
- 让执行链减少多轮 round-trip

### 回归类工具

- `run_regression_suite`

用途：

- 运行插件内最小自动回归
- 验证核心能力是否仍可用
- 适合在 UI 调整或执行链改动后快速冒烟

### 变更类工具

- `create_change_set`
- `get_change_set`
- `apply_change_set`

约束：

- 不允许绕过 ChangeSet 直接落盘
- 变更类工具必须配合 Review 页面使用

## Context Index

文件：`core/context/context_index_service.gd`

### 脚本符号索引

输入：

- `.gd`
- `.cs`

输出：

- `[{ kind, name, line }]`

用途：

- 快速定位脚本结构
- 提供类、方法、导出成员视图

### 场景依赖索引

输入：

- `.tscn`

输出：

- 资源依赖路径列表

用途：

- 快速理解场景依赖资源
- 让模型在修改场景相关资源前先拿到依赖上下文

## Context Bundle

`get_context_bundle` 会聚合：

- 当前编辑器上下文
- 指定脚本符号索引
- 指定场景依赖索引

意义：

- 一次调用获得结构化上下文
- 降低模型在只读阶段的多轮调用成本

## 最小自动回归

本地工具 `run_regression_suite` 当前用于：

- 启动后基础验证
- 手动烟测
- UI / 执行链改动后的快速回归

返回结构：

- `total`
- `passed`
- `failed`
- `items[]`

## 扩展判断标准

新增本地工具前先判断：

1. 该能力是否高频且通用。
2. 是否能被远端 MCP 稳定替代。
3. 是否涉及写入。
4. 是否值得为了本地实现引入额外 Godot 编辑器耦合。

若涉及写入，必须满足：

- 先生成 ChangeSet
- 再进入审批
- 不允许跳过 Review 页面直接应用
