# 分层职责

## Workbench Layer

当前主入口：`plugin.gd` 内部 `DockController` + `ui/orbit_dock.tscn`

职责：

- 渲染聊天页、Sessions 页、Tools 页、Review 页、Settings 页。
- 管理当前页切换、导航弱提示、面板样式和状态展示。
- 把用户动作转发给 `plugin.gd`：
  - 发送 prompt
  - 切换 provider
  - 执行工具
  - 审批 change set
  - 新建 / 打开 / 归档 / 删除 / 重命名 session
- 不直接访问网络，不直接写磁盘。

边界：

- 不负责 Provider 协议转换。
- 不负责工具实际执行。
- 不直接维护会话存储。

## Provider Layer

文件：`core/provider/*`

职责：

- 屏蔽不同模型供应商的协议差异。
- 统一返回结构：assistant message、tool calls、原始响应、流式状态。
- 处理 API Key 读取规则：本地配置优先，环境变量兜底。

当前提供：

- `openai_provider.gd`
- `anthropic_provider.gd`

说明：

- DeepSeek 当前通过 OpenAI Compatible 路径复用 `openai_provider.gd`。

## Tool Layer

文件：`core/tools/*`、`core/remote/remote_mcp_client.gd`

职责：

- 聚合本地工具与远端工具。
- 维护统一工具目录与工具来源前缀。
- 提供工具路由与执行分发。

组件：

- `LocalTools`：本地只读工具、上下文工具、ChangeSet 工具、回归工具。
- `RemoteMcpClient`：健康检查、工具发现、工具调用。
- `ToolCatalog`：统一目录。
- `ToolRouter`：本地 / 远端执行决策。

## Context Layer

文件：`core/context/context_index_service.gd`

职责：

- 提供脚本符号索引。
- 提供场景依赖索引。
- 支撑 `get_context_bundle` 聚合上下文。

边界：

- 只做结构化上下文抽取，不负责执行链决策。

## Execution Layer

文件：`core/execution/execution_service.gd`

职责：

- 执行 `analyze / propose / apply_with_approval`。
- 编排 Provider 与 ToolRouter。
- 向插件主控回调执行事件。
- 维护取消语义。

边界：

- 不直接更新 UI。
- 不直接写入会话存储。
- 不直接落盘，写入仍需经过 ChangeSet 流程。

## Change Layer

文件：`core/changes/change_set_service.gd`

职责：

- 创建 pending change set。
- 读取 diff 预览。
- 审批应用。
- 拒绝变更。
- 回滚最近一次已应用变更。

约束：

- 所有本地写入必须走 ChangeSetService。
- 不允许跳过审批直接落盘。

## Test Layer

文件：`core/testing/regression_runner.gd`

职责：

- 提供最小自动回归。
- 在启动期和手动工具执行时复用。
- 用于快速判断基础能力链是否仍然可用。

边界：

- 不替代完整集成测试。
- 默认不依赖远端 MCP。
